GeminiTruncationSettings
DocuGenious.Core.Configuration
nested under GeminiSettings · appsettings key: Gemini:Truncation
Controls exactly how much source data is sent to Gemini per request. All values are in characters (≈ tokens × 4). Increase limits for richer AI context on Tier 1; decrease to reduce token cost. Every field has a safe default so omitting the section from appsettings.json preserves previous behaviour.
PropertyTypeDefault (code)Current appsettings.jsonDescription
JiraDescriptionMaxCharsint20005000Max characters from each JIRA ticket description sent to Gemini. Increase for very detailed tickets.
JiraAcceptanceCriteriaMaxCharsint3001000Max characters per acceptance-criteria item (up to 10 items per ticket).
JiraCommentMaxCharsint3001000Max characters per JIRA comment body (up to 5 comments per ticket).
GitCommitMessageMaxCharsint2001000Max characters per Git commit message (up to 20 commits sent).
GitFileContentMaxCharsint20005000Max characters of source-file content per file (up to 20 files). Most impactful setting for code analysis quality.
PromptMaxCharsint80000100000Hard ceiling on the entire assembled prompt. Gemini 2.5 Flash Tier 1 supports up to ~1M tokens so this can safely be raised further for large repos.
✅
Latest values (as set in appsettings.json): File content 5,000 chars · JIRA description 5,000 chars · Comments/AC 1,000 chars each · Prompt ceiling 100,000 chars. These are ~2.5× the original defaults, giving significantly richer context without approaching Tier 1 quota limits.

DocuGenious.Integration.Gemini
Google Gemini AI integration via plain HttpClient REST calls. No SDK dependency.
class
GeminiService
DocuGenious.Integration.Gemini
implements IGeminiService
Core AI integration class. Builds prompts, calls the Gemini REST API, parses the response JSON through a 6-strategy repair pipeline, validates output quality, and returns a structured AnalysisResult. Handles rate limiting with automatic retry.
CallGeminiAsyncTask<string> — private
Low-level HTTP POST to /v1beta/models/{model}:generateContent. Reads candidates[0].content.parts[0].text from the response. On 429, embeds the retryDelay value from Gemini's JSON error body into the exception message for the caller to act on.
CallAndParseAsyncTask<AnalysisResult> — private
Orchestrates: truncate prompt → call Gemini → parse JSON → validate quality → retry on 429 (up to 3×) or 5xx (backoff: 1.5s / 4s / 9s). Only throws after all retries are exhausted.
ParseAnalysisResultAnalysisResult — private
Runs the 6-strategy JSON extraction pipeline on the raw AI text: ① raw text ② ```json fence ③ generic fence ④ outermost braces ⑤ truncated-JSON repair ⑥ nested-object flattening. Falls back to regex partial extraction if all fail.
ValidateOutputQualityvoid — private
Scores the result out of 100 points. Deducts points for empty or thin sections (e.g. executiveSummary < 30 words = −10 pts). Logs warnings but does not throw — a partial result is better than nothing.
BuildJiraContext / BuildGitContextstring — private instance
Assembles human-readable context strings from JiraTicket / GitRepositoryInfo objects. Applies all truncation limits read from _settings.Truncation: JiraDescriptionMaxChars, JiraAcceptanceCriteriaMaxChars, JiraCommentMaxChars, GitCommitMessageMaxChars, GitFileContentMaxChars — all fully configurable via appsettings.json.
NormalizeResult / RedistributeSectionsvoid — private static
NormalizeResult cleans placeholder text and empty arrays. RedistributeSections scans the executive summary for ## Heading markers and routes them to their correct AnalysisResult fields.
ExtractRetryDelay / ParseRetryAfterSecondsstring / double? — private static
Parses Gemini's error.details[].retryDelay JSON field (e.g. "4.197s") and embeds it in the exception message. ParseRetryAfterSeconds handles formats: [retry after Xs], Please retry in X.Xs., try again in Xm Y.Ys.
TruncateIfNeededstring — private instance
Hard-caps the full assembled prompt at PromptMaxChars characters (configurable via appsettings.json, default 80,000 / currently 100,000 chars ≈ 25,000 tokens) as a final safety net. Appends [Source data truncated] so the AI knows data was cut.
class
FlexibleApiEndpointListConverter  ·  FlexibleFeatureListConverter  ·  FlexibleStringListConverter
DocuGenious.Integration.Gemini
extends JsonConverter<T>
Tolerant JSON converters that handle the unpredictable output shapes that LLMs sometimes produce. Applied via JsonSerializerOptions during response deserialization.
ConverterProblem Solved
FlexibleApiEndpointListConverterHandles apiEndpoints as an array, a single object, a plain string, or a "GET /path – desc" formatted line
FlexibleFeatureListConverterHandles features as nested objects, a single item (not wrapped in array), or plain strings
FlexibleStringListConverterHandles dependencies/recommendations/knownIssues as arrays, newline-separated strings, or objects (flattened to "key: value" pairs)
⚠️
These converters exist because Gemini (like all LLMs) occasionally returns a field as a string when an array is expected, or nests objects one level too deep. Without them, deserialization would fail silently.

DocuGenious.Integration.Jira
Atlassian JIRA ticket fetching via SDK + direct REST API v3.
class
JiraService
DocuGenious.Integration.Jira
implements IJiraService
Fetches JIRA ticket data using a combination of the Atlassian .NET SDK (for issue fields and comments) and direct HTTP calls to JIRA REST API v3 (for subtasks — the SDK uses a deprecated v2 subtask endpoint). Extracts acceptance criteria from ticket descriptions using regex patterns.
GetTicketAsyncTask<JiraTicket>
Fetches one ticket. Calls Issues.GetIssueAsync() via SDK, then FetchSubTasksAsync() via REST v3. Extracts acceptance criteria via ExtractAcceptanceCriteria(). Wraps SDK errors with ExtractJiraApiError() for readable messages.
GetTicketsAsyncTask<List<JiraTicket>>
Fetches multiple tickets in sequence. Collects per-ticket exceptions and continues. Returns partial results. Throws only if every single ticket fails — prevents passing empty data to the AI.
FetchSubTasksAsyncTask<List<string>> — private
Calls POST /rest/api/3/issue/search with JQL parent = {key}. Parses the JSON response to extract subtask keys. Uses direct HTTP because the Atlassian SDK's subtask method targets the removed v2 endpoint.
ExtractAcceptanceCriteriaList<string> — private static
Regex-based extractor that finds acceptance criteria blocks in JIRA description text. Looks for sections headed "Acceptance Criteria", "AC:", or "Given/When/Then" patterns. Returns each criterion as a list item.
ExtractJiraApiErrorstring — private static
Parses the JSON error body embedded in Atlassian SDK exceptions. Extracts the errorMessages array for a readable error string instead of exposing raw HTTP bodies to the user.

DocuGenious.Integration.Git
Git repository cloning and analysis using LibGit2Sharp. Cloud-safe with automatic temp-dir cleanup.
class
GitService
DocuGenious.Integration.Git
implements IGitService
Clones remote repositories to a safe writable directory, analyzes commits, file contents, contributors, and detected technologies, then returns a GitRepositoryInfo object. Always cleans up the cloned directory in a finally block — even on error.
CloneAndAnalyzeAsyncTask<GitRepositoryInfo>
Resolves the clone base directory (configured path or system temp), creates a unique {repoName}_{8-char-guid} subdirectory, clones with PAT authentication, calls AnalyzeLocalRepositoryAsync(), then deletes the clone in a try/finally.
AnalyzeLocalRepositoryAsyncTask<GitRepositoryInfo>
Opens a local repository, optionally checks out a branch, then collects: last 30 commits, top 20 contributors, source files (filtered by extension, max 20, content capped at GitFileContentMaxChars), detected technologies, and a 2-level directory tree.
ValidateRemoteRepositoryAsyncTask<RepoValidationResult>
For github.com URLs, uses the GitHub REST API with Bearer auth to check accessibility and branch existence. For other URLs, uses LibGit2Sharp ls-remote. Returns details on default branch and whether the requested branch exists.
ResolveCloneBaseDirectorystring — private
Priority: ① configured CloneDirectory (if non-empty and writable) → ② Path.GetTempPath()/docugenious-repos. The fallback makes the service work on Azure App Service, Docker, and Linux without any configuration.
TryDeleteDirectoryvoid — private static
Resets FileAttributes.Normal on all files before deleting (LibGit2Sharp marks .git object files read-only). Then calls Directory.Delete(path, recursive: true).
DetectTechnologiesList<string> — private static
Maps file extensions to technology names (e.g. .cs → "C#", .ts → "TypeScript"). Also detects frameworks by well-known file names (e.g. Dockerfile → "Docker", *.csproj → ".NET").
ScanFilesList<GitFileInfo> — private
Walks source files filtered by DefaultExtensions. Skips SkippedDirectories (node_modules, .git, bin, obj, dist, etc.). Reads content only for text files under 100 KB. Returns at most 20 files.

PdfService
DocuGenious.Integration.Pdf
implements IPdfService
Renders an AnalysisResult into a branded, print-ready A4 PDF using QuestPDF's fluent API. Contains a full markdown block parser — it converts headings, bullets, numbered lists, code blocks, blockquotes, and inline bold/italic/code into styled QuestPDF elements.
GeneratePdfAsyncTask<string>
Entry point. Creates the output directory, sanitizes the file name, builds the QuestPDF Document (A4, 40px margin), calls header/content/footer composers, writes to disk, returns the full path.
ComposeContentvoid — private static
Conditionally renders each section (Executive Summary, Technical Overview, Architecture, Features, User Guide, Setup, Config, API Endpoints, Dependencies, Recommendations, Known Issues) only when the corresponding AnalysisResult field is non-empty.
RenderMarkdownvoid — private static
Parses raw markdown text into typed blocks (MdBlock: H1–H4, Paragraph, Bullet, Numbered, CodeBlock, HRule, BlockQuote) and renders each using matching QuestPDF styling. Falls back to JSON→markdown conversion if the field contains raw JSON.
RenderInlineTextvoid — private static
Handles inline markdown within a single paragraph: **bold**, *italic*, __bold__, _italic_, `code`. Uses a fast path (no spans) when no inline formatting is detected.
ComposeApiSectionvoid — private static
Renders the API endpoints table with color-coded HTTP verb badges (GET=green, POST=blue, PUT=orange, PATCH=purple, DELETE=red). Expands request/response bodies in dark terminal blocks below each row.
ConvertJsonToMarkdownstring — private static
Safety net: if an AnalysisResult field contains valid JSON (e.g. the AI wrapped its response in a nested object), converts it to readable markdown prose rather than rendering raw JSON as a code block.

JobStatusService
DocuGenious.Api.Services
In-memory store for tracking the real-time status of in-flight generation jobs. The Blazor UI polls /api/documentation/status/{jobId} every 1.5 seconds to update the spinner message shown to the user. Entries expire after 30 minutes to prevent memory leaks.
Updatevoid
Creates or updates the status for a given jobId. Resets the 30-minute expiration window on every update. Parameters: string jobId string status
Getstring?
Returns the current status string, or null if the job ID is unknown or the entry has expired.
Removevoid
Explicitly removes a job entry when generation completes. Keeps memory clean rather than waiting for expiry.
PurgeExpiredvoid
Scans the ConcurrentDictionary and removes all entries whose ExpiresAt has passed. Called opportunistically on Get() to avoid unbounded growth.

DocuGenious.Blazor — Pages
The single-page Blazor UI at route /.
Blazor Component
Home.razor
DocuGenious.Blazor.Pages  ·  Route: /
The entire DocuGenious web UI in a single Razor component. Manages the complete generation workflow from form input through to PDF download. Uses a modal dialog with 4 states (Validating, Generating, Success, Error).
Field / PropertyPurpose
_sourceType, _docType, _repoUrl, _branch, _additionalContext, _outputFileNameBound to form inputs. _docType defaults to "TechnicalDocumentation".
_ticketInput, _ticketIdsTag-input pattern: user types in _ticketInput, presses Enter/comma to add to _ticketIds list
_state (UiState enum)Controls which modal panel is shown: Form / Validating / Generating / Success / Error
CanSubmitComputed property: source type selected + tickets entered or typed + repo URL filled → enables the Generate button
StartValidationasync Task
Flushes any typed-but-not-confirmed ticket from _ticketInput, opens the validation dialog, runs ValidationService.ValidateAsync() with live row updates, then calls Generate() if all checks pass.
Generateasync Task
Posts the GenerateRequest to the API with a unique JobId, starts the status polling loop (PollStatusAsync) in the background, awaits the result, then transitions to Success or Error state.
PollStatusAsyncasync Task
Runs every 1.5 seconds while generation is in progress. Calls GetJobStatusAsync() and updates _statusMessage to drive the spinner text. Cancelled via CancellationTokenSource when generation completes.
CloseDialog / ResetFormvoid
CloseDialog hides the modal. Calls ResetForm() on Success or validation failure (fresh start), but not on Error (keeps inputs so user can fix and retry). ResetForm resets all fields — _docType resets to "TechnicalDocumentation".

class
CliHandler
DocuGenious.Console.Cli
The main CLI orchestrator. Provides an interactive Spectre.Console loop where users can generate documents, see live progress spinners, and generate multiple documents in one session. Uses IGroqService (Groq AI) rather than Gemini — independent of the API project's AI choice.
RunAsyncTask
Main loop: print banner → validate services → collect request → process → ask "generate another?" → repeat. Handles top-level exceptions with formatted Spectre error panels.
ValidateServicesAsyncTask
Shows a live spinner ("Connecting to Gemini AI...") while calling ValidateConnectionAsync(). Exits the process with code 1 if the AI service is unreachable.
CollectRequestAsyncTask<DocumentationRequest>
Interactive prompts using Spectre.Console SelectionPrompt and TextPrompt. Collects source type, JIRA ticket IDs, Git URL and branch, document type, and output file name.
ProcessRequestAsyncTask
Executes the full pipeline with a live progress display. Fetches JIRA, clones/analyzes Git, calls Groq AI, generates PDF. Shows each step completing in the terminal with timing information.
